# Benchmark: Speed-Optimised MARS Variants vs Original MARS

Comparison of **alignment quality** and **runtime** of the speed-optimised MARS
presets (`-Q 1/2/3`) against the original implementation (`-Q 0`, bit-faithful to
Ayad & Pissis 2017), on the **same datasets the original authors used**, plus
larger synthetic sets that expose where the optimisations pay off.

Generated by `run_bench.py` / `analyze.py` (raw data in `out/timing.csv`,
`out/quality.csv`). Machine: 16-core shared Linux box; MARS `g++ -O3`; MSA
program ClustalΩ 1.2.4.

---

## 1. Method

### 1.1 Timing protocol (soundness)
Each MARS run is **repeated 3×**; the reported runtime is the **mean**, and the
per-cell spread `(max−min)/mean` is checked. **0 of 72 cells exceeded 25%
spread** — despite this being a shared machine with visible load fluctuation
(load average ranged 0.2–12 during the run), MARS timings were highly
reproducible (e.g. `n=500 -Q 0`: 1302.73 / 1302.78 / 1301.61 s, spread 0.09%).
ClustalΩ is deterministic, so it is run once per cell for the quality measure.

### 1.2 Datasets
**Paper datasets** (cloned from `github.com/lorrainea/MARS`):
- **Synthetic** (`exp1/`): 9 INDELible/JC69 sets `A.B.C` = (#seqs × avg-bp ×
  %-dissimilarity): `{12,25,50}.2500.{5,20,35}`, fed to MARS as the randomly
  rotated `.rot.fas` input.
- **Real** (`exp2/`): `Mammals` (12 mtDNA, ~16.8 kbp), `Primates` (16 mtDNA,
  ~16.6 kbp), `Viroids` (18 RNA, ~363 bp).

**Scaled sets** (`gen_circular.py`, hierarchical-tree circular simulation with
random linearisation + ground-truth `.orig` sidecar): `n ∈ {100,250,500}`,
**m = 2000**, divergence `med` (~8%) and `high` (~16%).

**MARS parameters** per the paper: synthetic `q=5 -l 50 -P 1.0`;
Mammals/Primates `q=5 -l 100 -P 2.0`; Viroids `q=4 -l 25 -P 1.0`; scaled
`q=5 -l 50 -P 1.0`. Presets: `-Q 0`=original, `-Q 1`=`-N -g1` (safe),
`-Q 2`=`-G5 -N -g1` (balanced), `-Q 3`=`-G1 -N -g1` (fastest).

### 1.3 Pipeline & metrics
`refine (mars -Q L)` → `align (clustalo --outfmt=phy)` → `measure`. ClustalΩ is
given a **30-min cap**; with stable load every cell completed (worst 723 s), so
**ground-truth AVPD is obtained for all 90 cells — no timeouts**. Baseline =
ClustalΩ on unrefined input (no MARS).

- **AVPD** = mean over all unordered pairs of #alignment columns where the two
  sequences differ (gap counts as difference) — the paper's headline metric.
- **circvar** (scaled only) = circular frame-consistency of chosen rotations vs
  ground truth (0 = perfect, ~0.95 = random).
- **rotQ0** (scaled only) = rotation-structure agreement with `-Q 0` up to the
  arbitrary global frame (1.0 = identical).

**Calibration gate (passed).** AVPD formula reproduces the paper's Table 5
*exactly* on the authors' own MUSCLE alignments: Mammals 5421.04 (paper 5421),
Primates 4070.98 (4070), Viroids 221.34 (221).
**Reproduction gate (passed).** `-Q 0` reproduces the authors' `.mars.fas`
refined outputs exactly (12/12, 50/50, 12/12) → our build ≡ original MARS.

pay the Θ(n·m²) progressive-alignment cost (shared overhead), so the ratio
reflects only the Θ(n²) refine + Θ(n³) guide-tree savings: ~2–6× at paper scale
(n≤50), rising to **~25–65× at n=500** (`-Q 0` 1325 s → `-Q 3` 21 s).

At moderate divergence every preset matches `-Q 0` closely (AVPD within ~1.5%,

tree + single-reference rotation voting loses the frame for a minority of
sequences in the largest, most divergent set. `-Q 2` and `-Q 3` stay closer to
`-Q 0` there. `-Q 0` itself is not perfect in that regime either.

## 5. Findings

1. **`-Q 1` (safe preset) is quality-faithful everywhere**: within 0.8% AVPD of
   original MARS on all 12 paper datasets; circvar ≈ 0.0001 and rotQ0 ≈ 1.000 up
   to n=500 at moderate divergence. Recommended default.
2. **`-Q 2/3` (aggressive) preserve quality on synthetic + moderate-divergence
   data** but drift 5–10% on the hard real genomes (Mammals/Primates/Viroids)
   and ~5% on `n500.med -Q 2`; they remain the fastest.
3. **Speedup scales with n** while timing stays reproducible (0/72 high-variance
   cells): ~2–6× at paper scale (n≤50) → **~25–65× at n=500, m=2000**.
4. **All presets retain MARS's benefit over no-MARS** on every dataset (AVPD far
   below the unrefined baseline) — the rotation refinement still helps.

## 6. Limitations
- Single downstream MSA program (ClustalΩ 1.2.4); absolute AVPD differs from the
  2017 paper's ClustalΩ version, but the AVPD *formula* is exact-validated
  against the authors' MUSCLE alignments and all presets use the same program,
  so comparisons are internally controlled.
- No phylogenetic RF-distance or TCS (ClustalΩ-only scope); AVPD is the sole
  quality metric, consistent with the paper's Table 6 time-accuracy measure.
- Runtime measured on a shared 16-core machine. Load fluctuated (0.2–12) during
  the run, but the 3-rep mean was stable (max spread <1% on the expensive cells,
  0/72 cells flagged) — see §1.1.
- `-Q 2/3` quality on highly-divergent real mtDNA and `n500.high` is measurably
  below `-Q 0`; users in that regime should prefer `-Q 1`.
